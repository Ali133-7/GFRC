## 📋 تحليل المشاكل الشائعة

### المشكلة 1: القيمة فارغة (null)

**الأعراض:**
```
القيمة_الفعلية: "لا شيء (null)"
القيمة_موجودة: "لا ❌"
```

**الأسباب المحتملة:**
1. ❌ الحقل **غير إلزامي** والمستخدم لم يدخل قيمة
2. ❌ الحقل في **خطوة أخرى** (Step 2+)
3. ❌ الحقل **مخفي** أو معطل في الـ Form
4. ❌ اسم الحقل في الـ Rule **لا يطابق** اسم الحقل الفعلي

**الحلول:**

#### الحل 1: اجعل الحقل إلزامي
```
في مصمم سير العمل:
1. افتح تبويب الحقول
2. ابحث عن "سجلات الدلالين"
3. ضع علامة ✅ في "إلزامي"
4. احفظ
```

#### الحل 2: أضف قيمة افتراضية
```
في مصمم سير العمل:
1. افتح تبويب الحقول
2. ابحث عن "سجلات الدلالين"
3. أدخل "0" في "القيمة الافتراضية"
4. احفظ
```

#### الحل 3: تحقق من خطوة الحقل
```sql
-- في قاعدة البيانات
SELECT 
    wf.label_override,
    ws.title_ar as الخطوة,
    wf.is_required as إلزامي
FROM workflow_fields wf
LEFT JOIN workflow_steps ws ON wf.step_id = ws.id
WHERE wf.register_field_id = 'اسم_الحقل'
   OR wf.id = 'معرف_الحقل';
```

**النتيجة المتوقعة:**
```
الخطوة: "البيانات الأساسية" ✅
إلزامي: 1 ✅
```

---

### المشكلة 2: الحقل في خطوة خاطئة

**الأعراض:**
```
الخطوة: "الخطوة 2"
القيمة_الفعلية: "لا شيء (null)"
```

**السبب:**
الحقل موجود في Step 2، لكن القاعدة تُنفذ في Step 1!

**الحل:**
```
في مصمم سير العمل:
1. افتح تبويب الحقول
2. ابحث عن الحقل
3. غيّر "الخطوة" إلى Step 1
4. احفظ
```

---

### المشكلة 3: معرف الحقل خاطئ في القاعدة

**الأعراض:**
```
اسم_الحقل: "حقل مخصص (custom_XXX...)"
الخطوة: "الحقل غير موجود في هذه النسخة"
```

**السبب:**
القاعدة تشير إلى معرف حقل قديم (من نسخة مستنسخة)!

**الحل:**
```
1. احذف القاعدة القديمة
2. أنشئ قاعدة جديدة
3. اختر الحقل من القائمة (سيتم استخدام المعرف الصحيح)
4. احفظ القاعدة
```

**أو للاستنساخ:**
```
تم إصلاح مشكلة الاستنساخ!
القواعد الآن تُحدّث معرفاتها تلقائياً عند الاستنساخ.
```

## 📊 ملخص وضع التصحيح

### المعلومات المتوفرة:

| المعلومة | الوصف |
|----------|-------|
| **اسم_القاعدة** | اسم القاعدة التي تُقيّم |
| **اسم_الحقل** | اسم الحقل من الـ Label |
| **معرف_الحقل** | المعرف الفعلي (UUID أو register_field_id) |
| **الخطوة** | أي خطوة يحتويها الحقل |
| **العامل** | العامل المقارن بالعربية |
| **القيمة_المتوقعة** | القيمة في الشرط |
| **القيمة_الفعلية** | القيمة المُرسلة من الـ Form |
| **القيمة_موجودة** | هل القيمة موجودة؟ ✅/❌ |
| **النتيجة** | هل الشرط محقق؟ نعم/لا |

## 🔧 نصائح للاستخدام الفعال

### 1. **دائماً شغّل logs أثناء التطوير:**
```bash
tail -f backend/storage/logs/laravel.log
```

### 2. **استخدم أسماء حقول واضحة:**
```
✅ سجلات الدلالين
❌ حقل 1
```

### 3. **اجعل الحقول المهمة إلزامية:**
```
✅ is_required: true
❌ is_required: false
```

### 4. **أضف قيم افتراضية:**
```
✅ default_value: "0"
❌ default_value: null
```

### 5. **تحقق من خطوة الحقل:**
```
تأكد أن الحقل في نفس الخطوة التي تُنفذ فيها القاعدة
```
