## 🎯 المشكلة

**الأعراض:**
```
[SKIP] احتساب قيمة السجلات
Conditions: عدد السجلات greater_than "0" [actual="null"]
```

**المشكلة:** المستخدم أدخل قيمة في حقل "عدد السجلات" لكن القيمة **لم تُرسَل** من الـ Form!

### الخطوة 4: حلّل المشكلة

**إذا لم تجد الحقل في `values`:**

**الأسباب المحتملة:**

#### 1. ❌ الحقل غير مرئي
```javascript
// في Console
document.querySelector('input[name="custom_5d7155cb-..."]');
// النتيجة: null → الحقل غير موجود في الـ DOM!
```

**الحل:**
- تحقق من أن الحقل في الخطوة الحالية (Step 1)
- تحقق من أن الحقل غير مخفي بقاعدة

---

#### 2. ❌ الحقل معطل (Disabled)
```javascript
// في Console
const field = document.querySelector('input[name="custom_5d7155cb-..."]');
console.log('Disabled:', field?.disabled);
// النتيجة: true → الحقل معطل!
```

**الحل:**
- تحقق من أن الحقل غير `readonly` أو `locked`
- تحقق من أن الحقل ليس `disabled`

---

#### 3. ❌ الحقل في خطوة أخرى
```
في Rule Execution Trace:
عدد السجلات (custom_5d7155cb-...): visible=Y, step=2
```

**الحل:**
- غيّر خطوة الحقل إلى Step 1
- أو نفّذ القاعدة في Step 2

---

#### 4. ❌ الحقل غير إلزامي والمستخدم لم يدخل قيمة
```
في Console:
values: { ... }
// لا يوجد custom_5d7155cb-...
```

**الحل:**
- اجعل الحقل **إلزامي** ✅
- أو أضف **قيمة افتراضية** (مثلاً: 0)

---

### الخطوة 5: راقب Backend Logs

**في Terminal:**
```bash

tail -f storage/logs/laravel.log | grep -i "استلام\|تنظيف\|تطبيع"
```

**سترى رسائل مثل:**

```
  }
```

#### 2. تنظيف القيم:

```
  "field_id": "custom_5d7155cb-...",
```

#### 4. تحذير: حقل غير موجود
```

{
  "field_id": "custom_5d7155cb-...",
  "value": "2"
}
```

**المعنى:** الحقل مُرسَل من الـ Form لكن **ليس في `stepFields`**!

**الحل:**
- تحقق من أن `step_id` للحقل صحيح
- تحقق من أن الحقل في الخطوة الحالية

---

## 📋 قائمة التحقق

### قبل إرسال الـ Form:

- [ ] الحقل **موجود** في الـ DOM
- [ ] الحقل **غير معطل** (disabled=false)
- [ ] الحقل **غير للقراءة فقط** (readonly=false)
- [ ] الحقل **في الخطوة الحالية** (step_id صحيح)
- [ ] الحقل **مرئي** (is_visible=true)
- [ ] المستخدم **أدخل قيمة**
- [ ] القيمة **موجودة في `values` state**

### بعد إرسال الـ Form:

- [ ] القيمة **موجودة في Console log**
- [ ] Backend **استلم القيمة**
- [ ] Sanitize **لم يتجاهل القيمة**
- [ ] Normalize **طبّق القيمة على جميع aliases**
- [ ] Rule **وجدت القيمة** (actual ليس null)

```
في مصمم سير العمل:
1. تبويب الحقول
```

```
3. أدخل "0" في "القيمة الافتراضية"
4. احفظ
```

### الحل 3: تحقق من خطوة الحقل

```sql
-- في قاعدة البيانات
SELECT 
    label_override as "اسم الحقل",
    step_id,
    ws.title_ar as "الخطوة"
FROM workflow_fields wf
LEFT JOIN workflow_steps ws ON wf.step_id = ws.id
WHERE label_override = 'عدد السجلات';
```

**النتيجة المتوقعة:**
```
الخطوة: "البيانات الأساسية" ✅
```

### الحل 4: تحقق من Field States

**في Rule Execution Trace:**
```
عدد السجلات (custom_5d7155cb-...): visible=Y, required=N, readonly=N
```

**يجب أن يكون:**
- `visible=Y` ✅
- `readonly=N` ✅
- `required=Y` (مستحسن) ✅

## 📊 ملخص

| المرحلة | ما يجب التحقق منه |
|---------|-------------------|
| **Frontend** | الحقل موجود، غير معطل، القيمة في state |
| **Console** | القيمة في `values` object |
| **Backend** | القيمة مُستلمة، مُنظّفة، مُطبّعة |
| **Rule** | `actual` ليس `null` |
